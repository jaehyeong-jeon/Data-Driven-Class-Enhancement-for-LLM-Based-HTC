## Preprocessing: Build Category Descriptions

Before running experiments with `--use_desc`, generate category descriptions for each dataset:

```bash
python build_descriptions.py --dataset wos     --model gpt-4o-mini
python build_descriptions.py --dataset amazon  --model gpt-4o-mini
python build_descriptions.py --dataset dbpedia --model gpt-4o-mini
```

Descriptions are saved to `dataset/{dataset}/description.json`.

**Data files:** Full training sets (e.g. `train_40k.csv`, `Data.xlsx`) are gitignored due to size.
Each dataset includes a `*_sample.csv` with 20 documents per class, which `build_descriptions.py`
uses automatically if the full file is not present.

**How descriptions are generated (three-stage pipeline):**

1. **Leaf** вЂ” For each leaf class, up to `--sample_n` (default: 10) documents are sampled.
   Each document is passed to an LLM to extract *keywords* and a *summary*.
   All extractions are then synthesized into a single class description:
   `{definition, characteristics, core_keywords}`.

2. **Parent** вЂ” Working bottom-up through the hierarchy, each parent node's description
   is synthesized from its children's descriptions (no documents involved).

3. **Discriminative** вЂ” Sentence embeddings are computed for all class descriptions per layer.
   For each class, its most similar neighbours (cosine similarity в‰Ґ `--sim_threshold`) are
   identified, and an LLM generates a `detail_description` that explicitly distinguishes the
   class from those confusable neighbours.

**Resuming / partial runs:**

| Flag | Effect |
|------|--------|
| `--resume` | Skip already-completed entries |
| `--parents_only` | Skip leaf step; redo parents + discriminative |
| `--discriminative_only` | Only redo discriminative step |

**Key options:**

| Argument | Default | Description |
|----------|---------|-------------|
| `--sample_n` | `10` | Docs sampled per leaf node |
| `--sim_threshold` | `0.5` | Cosine similarity threshold for confusable neighbours |
| `--sim_drop_ratio` | `0.8` | Drop neighbours below `top1_score Г— ratio` |

## Methods

### Baselines

| Method | Description |
|--------|-------------|
| `flatten` | Predict leaf label directly from all candidates |
| `path` | List all rootв†’leaf paths, pick one |
| `bfs` | BFS layer-by-layer with LLM |
| `dfs` | DFS with backtracking |
| `parent` | Layer-by-layer conditioned on parent prediction |
| `ensemble` | MV + PVV over flatten/path/bfs/dfs |

### Ours

| Method | Description |
|--------|-------------|
| `all_in_one` | Single-call path selection |
| `topdown_llm_beam` | Top-down beam search with LLM ranker |
| `pointwise` | Parallel yes/no per path в†’ final selection |

## Usage

```bash
# Single method
python main.py --dataset amazon --model gpt-4o-mini --method flatten
python main.py --dataset amazon --model gpt-4o-mini --method topdown_llm_beam --beam_size 3
python main.py --dataset wos    --model gpt-4o-mini --method pointwise --use_desc

# Run all baselines / all ours at once
python main.py --dataset amazon --model gpt-4o-mini --method all_baseline
python main.py --dataset amazon --model gpt-4o-mini --method all_ours

# Ensemble
python main.py --dataset wos --model gpt-4o-mini --method ensemble

# Segment (e.g. first 500 samples)
python main.py --dataset amazon --model gpt-4o-mini --method flatten --segment 0,500
```

- `eval5k.txt` вЂ” per-sample predictions
- `eval5k_scores.json` вЂ” accuracy / macro-F1 per layer
- `eval5k_details.jsonl` вЂ” per-sample LLM responses (ours methods only)
