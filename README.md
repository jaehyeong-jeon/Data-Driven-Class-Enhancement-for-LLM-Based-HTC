## Preprocessing: Build Category Descriptions

Pre-built `description.json` files are already included in the repo for all three datasets, so this step can be skipped unless you want to regenerate them.

```bash
python build_descriptions.py --dataset wos     --model gpt-5.4-mini
python build_descriptions.py --dataset amazon  --model gpt-5.4-mini
python build_descriptions.py --dataset dbpedia --model gpt-5.4-mini
```

Descriptions are saved to `dataset/{dataset}/description.json`.

**Data files:** Full training sets (e.g. `train_40k.csv`, `Data.xlsx`) are gitignored due to size.
Each dataset includes a `*_sample.csv` with 20 documents per class, which `build_descriptions.py`
uses automatically if the full file is not present.

**How descriptions are generated (three-stage pipeline):**

1. **Leaf** — For each leaf class, up to `--sample_n` (default: 10) documents are sampled.
   Each document is passed to an LLM to extract *keywords* and a *summary*.
   All extractions are then synthesized into a single class description:
   `{definition, characteristics, core_keywords}`.

2. **Parent** — Working bottom-up through the hierarchy, each parent node's description
   is synthesized from its children's descriptions (no documents involved).

3. **Discriminative** — Sentence embeddings are computed for all class descriptions per layer.
   For each class, its most similar neighbours (cosine similarity ≥ `--sim_threshold`) are
   identified, and an LLM generates a `detail_description` that explicitly distinguishes the
   class from those confusable neighbours.

**Resuming / partial runs:**

| Flag | Effect |
|------|--------|
| `--resume` | Skip already-completed entries |
| `--parents_only` | Skip leaf step; redo parents + discriminative |
| `--discriminative_only` | Only redo discriminative step |

**Key options:**

| Argument | Default | Description |
|----------|---------|-------------|
| `--sample_n` | `10` | Docs sampled per leaf node |
| `--sim_threshold` | `0.5` | Cosine similarity threshold for confusable neighbours |
| `--sim_drop_ratio` | `0.8` | Drop neighbours below `top1_score × ratio` |

## Methods

### Baselines

| Method | Description |
|--------|-------------|
| `flatten` | Predict leaf label directly from all candidates |
| `path` | List all root→leaf paths, pick one |
| `bfs` | BFS layer-by-layer with LLM |
| `dfs` | DFS with backtracking |
| `parent` | Layer-by-layer conditioned on parent prediction |
| `ensemble` | MV + PVV over flatten/path/bfs/dfs |

### Ours

| Method | Description |
|--------|-------------|
| `all_in_one` | Single-call path selection |
| `topdown_beamsearch` | Top-down beam search with LLM ranker |
| `pointwise` | Parallel yes/no per path → final selection |

## Usage

```bash
# Single method
python main.py --dataset amazon --model gpt-5.4-mini --method flatten
python main.py --dataset amazon --model gpt-5.4-mini --method topdown_beamsearch --beam_size 3
python main.py --dataset wos    --model gpt-5.4-mini --method pointwise --use_desc

# Run all baselines / all ours at once
python main.py --dataset amazon --model gpt-5.4-mini --method all_baseline
python main.py --dataset amazon --model gpt-5.4-mini --method all_ours

# Ensemble
python main.py --dataset wos --model gpt-5.4-mini --method ensemble

# Segment (e.g. first 500 samples)
python main.py --dataset amazon --model gpt-5.4-mini --method flatten --segment 0,500
```

## Key Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--dataset` | — | `amazon` / `dbpedia` / `wos` |
| `--model` | — | OpenAI model name |
| `--method` | — | See methods table above |
| `--beam_size` | `3` | Beam width for `topdown_beamsearch` |
| `--selection_mode` | `all_in_one` | `per_child` / `all_in_one` / `pointwise` |
| `--use_desc` | `False` | Append category descriptions to prompts |
| `--max_workers` | `10` | Parallel LLM call threads |
| `--segment` | `None` | `start,end` index slice |

## Results

Saved under `results/{dataset}/{baseline|ours}/{only_class_name|use_desc}/{method}/{model}/`.

Each run produces:
- `eval5k.txt` — per-sample predictions
- `eval5k_scores.json` — accuracy / macro-F1 per layer
- `eval5k_details.jsonl` — per-sample LLM responses (ours methods only)
