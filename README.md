# LLM-Based Hierarchical Text Classification

This repository contains experiments for LLM-based hierarchical text classification (HTC) on three benchmark datasets: **Amazon**, **DBpedia**, and **WOS**.

---

## Quick Start

```bash
pip install -r requirements.txt
cp .env.example .env   # add your OPENAI_API_KEY
```

Run an experiment:

```bash
python main.py --dataset amazon --model gpt-5.4-mini --method flatten
python main.py --dataset wos    --model gpt-5.4-mini --method topdown_beamsearch --use_desc
```

## Methods

### Baselines

| Method | Description |
|--------|-------------|
| `flatten` | Predict leaf label directly from all candidates |
| `path` | List all rootв†’leaf paths, pick one |
| `bfs` | Layer-by-layer selection (BFS order) |
| `dfs` | Depth-first traversal with backtracking |
| `parent` | Each layer conditioned on the parent's prediction |
| `ensemble` | Majority vote over flatten / path / bfs / dfs |

### Ours

| Method | Description |
|--------|-------------|
| `all_in_one` | Single LLM call to select the full path |
| `topdown_beamsearch` | Top-down beam search with LLM scoring at each layer |
| `pointwise` | Parallel yes/no scoring per candidate path, then final selection |

## Usage

```bash
# Single method
python main.py --dataset amazon --model gpt-5.4-mini --method flatten
python main.py --dataset amazon --model gpt-5.4-mini --method topdown_beamsearch --beam_size 3
python main.py --dataset wos    --model gpt-5.4-mini --method pointwise --use_desc

# Run all baselines or all ours at once
python main.py --dataset amazon --model gpt-5.4-mini --method all_baseline
python main.py --dataset amazon --model gpt-5.4-mini --method all_ours

# Ensemble
python main.py --dataset wos --model gpt-5.4-mini --method ensemble

# Run on a subset (e.g. first 500 samples)
python main.py --dataset amazon --model gpt-5.4-mini --method flatten --segment 0,500
```

Output files are saved under `results/{dataset}/{baseline|ours}/{only_class_name|use_desc}/{method}/{model}/`.

- `eval5k_scores.json` вЂ” accuracy / macro-F1 per layer
- `eval5k_details.jsonl` вЂ” per-sample LLM responses (ours methods only)

## Category Descriptions (`--use_desc`)

Pre-built `description.json` files are already included for all three datasets вЂ” **no need to regenerate unless you want to**.

Each description is generated by `build_descriptions.py` via a three-stage pipeline:

1. **Leaf** вЂ” For each leaf class, up to `--sample_n` documents are sampled. An LLM extracts *keywords* and a *summary* from each, which are then synthesized into `{definition, characteristics, core_keywords}`.
2. **Parent** вЂ” Parent node descriptions are synthesized bottom-up from their children's descriptions.
3. **Discriminative** вЂ” For classes with high embedding similarity, an LLM generates a `detail_description` that explicitly distinguishes them from their confusable neighbours.

To regenerate:

```bash
python build_descriptions.py --dataset wos     --model gpt-5.4-mini
python build_descriptions.py --dataset amazon  --model gpt-5.4-mini
python build_descriptions.py --dataset dbpedia --model gpt-5.4-mini
```

| Flag | Effect |
|------|--------|
| `--resume` | Skip already-completed entries |
| `--parents_only` | Skip leaf step; redo parents + discriminative |
| `--discriminative_only` | Only redo discriminative step |

| Argument | Default | Description |
|----------|---------|-------------|
| `--sample_n` | `10` | Docs sampled per leaf node |
| `--sim_threshold` | `0.5` | Cosine similarity threshold for confusable neighbours |
| `--sim_drop_ratio` | `0.8` | Drop neighbours below `top1_score Г— ratio` |
